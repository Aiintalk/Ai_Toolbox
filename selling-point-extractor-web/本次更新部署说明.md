# 产品卖点提取器 — 本次更新部署说明

> 服务地址：http://121.40.174.53/selling-point-extractor
> 服务器路径：`/var/www/selling-point-extractor-web`
> PM2 服务名：`selling-point-extractor`
> 端口：3011

---

## 本次更新内容

1. **新增 .pages 文件上传支持**：用户可以直接上传 Apple Pages 格式文件，系统自动解析文本内容
2. **新增「和 AI 聊聊」对话框**：分析完成后，用户可以和 AI 讨论调整卖点，直到满意为止
3. **新增「最终卖点卡」区块**：自动从分析结果中提取极致卖点卡，支持一键复制和保存为文件下载
4. **移除「去写千川脚本」跳转按钮**

---

## 涉及修改的文件

| 文件 | 改动说明 |
|------|----------|
| `package.json` | 新增两个依赖：`jszip`、`snappyjs` |
| `app/page.tsx` | 前端页面：文件上传支持 .pages、底部新增卖点卡展示+保存功能 |
| `app/api/parse-file/route.ts` | 后端接口：新增 .pages 文件解析逻辑 |

---

## 新增依赖

```
jszip: ^3.10.1      — 解压 .pages 文件（本质是 zip 包）
snappyjs: ^0.7.0    — 解压 .pages 内部的 IWA 数据（snappy 压缩格式）
```

---

## 部署步骤

```bash
# 1. 进入项目目录
cd /var/www/selling-point-extractor-web

# 2. 备份当前版本（可选但建议）
cp -r app app.bak

# 3. 上传/更新以下三个文件（从本地代码库同步）
#    - package.json
#    - app/page.tsx
#    - app/api/parse-file/route.ts

# 4. 安装新依赖
npm install

# 5. 构建
npm run build

# 6. 重启服务
pm2 restart selling-point-extractor

# 7. 验证服务正常
pm2 status selling-point-extractor
curl -s http://127.0.0.1:3011/selling-point-extractor | head -5
```

## 回滚方案

如果部署出现问题：

```bash
# 恢复备份
cd /var/www/selling-point-extractor-web
rm -rf app
mv app.bak app

# 重新构建和重启
npm run build
pm2 restart selling-point-extractor
```
