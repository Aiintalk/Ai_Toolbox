# Portal — AI 工具箱导航页

## 1. 项目定位

作为整个 AI 工具箱 Monorepo 的入口导航页，以静态 HTML 的形式集中展示所有工具的访问入口和上线状态。

## 2. 当前功能

- 展示工具箱中所有工具的卡片列表（含上线与开发中状态）
- 支持点击直接跳转到已上线工具（新标签页打开）
- 工具按分类分区展示：内容仿写 / 复盘工具 / 底层通用
- 未上线工具以半透明禁用样式展示，防止误点击

**当前已上线工具（2 个）：**

| 工具 | 分类 | 访问地址 |
|------|------|----------|
| 人设脚本仿写助手 | 内容仿写 | `/persona-writer/` |
| 对标分析助手 | 底层通用 | `/benchmark-analyzer/` |

**开发中工具（6 个）：**
- 千川脚本仿写助手
- 直播间脚本仿写助手
- 人设脚本复盘助手
- 千川脚本复盘助手
- 直播间脚本复盘助手
- 人设定位助手

## 3. 使用流程

1. 打开浏览器访问部署地址（如 `http://192.168.74.130/`）
2. 在导航页中找到目标工具卡片
3. 点击已上线的卡片跳转至对应工具页面
4. 灰显卡片为开发中，暂不可点击

## 5. 核心接口 / 核心模块

本项目为纯静态页面，无后端接口。核心结构如下：

- **`.header`**：顶部标题栏，展示工具箱名称和工具总数
- **`.section`**：主内容区，按分类组织卡片网格
- **`.card`**：单个工具卡片，包含图标、标题、描述、标签
- **`.card-disabled`**：开发中工具的禁用样式（`pointer-events: none`）

## 6. 环境变量 / 运行要求

- 无任何运行时依赖，无需 Node.js 或构建工具
- 直接以静态文件方式部署（Nginx / Apache / 对象存储均可）
- 工具卡片跳转地址当前硬编码为内网 IP `192.168.74.130`，上线前需按实际部署地址修改

## 7. 开发注意事项

- 新增工具时，在 `index.html` 对应分类的 `.grid` 中添加卡片 HTML
- 已上线工具使用 `<a class="card ...">` 标签，带 `href` 和 `target="_blank"`
- 开发中工具使用 `<div class="card ... card-disabled">` 标签，无链接
- 卡片颜色主题：内容仿写用蓝（`card-write`）、复盘工具用橙（`card-review`）、底层通用用紫（`card-base`）
- 修改工具总数时，同步更新 `.header p` 中的"共 N 个工具"文案
- 本页面无 JS，样式全部内联在 `<style>` 中，不要引入外部依赖
