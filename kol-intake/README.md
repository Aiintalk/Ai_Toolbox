# kol-intake — 红人信息采集助手

## 1. 项目定位

面向 MCN / 红人孵化团队的 KOL 入职信息采集工具。通过拟人化 AI 对话引导红人完成约 23 道结构化问题，结束后自动生成 AI 分析报告，并提供管理后台供团队查阅和下载档案。

## 2. 当前功能

- AI 对话式问卷，以聊天气泡形式逐题引导，顶部实时显示进度条
- 支持文本、长文本、多条循环收集（输入"没了"结束）三种题型
- 每道题回答后 AI 生成自然过渡语（bridge）
- 提交后调用云雾 AI（`claude-opus-4-6` + extended thinking）自动生成人格分析报告
- 报告在线展示，支持一键下载为 `.txt` 文件
- 管理后台（`/admin`）展示所有提交记录，可查看问答详情和分析报告
- 管理员可对任意提交导出 `.docx` 格式信息采集表
- 提交数据以 JSON 文件存储在服务器本地 `data/` 目录

## 3. 使用流程

**红人端**
1. 访问 `/kol-intake/`，点击「开始聊吧」
2. 按题目逐条回答；多条收集类题目输入「没了」结束该题
3. 全部完成后点击「提交并生成分析报告」
4. 等待 AI 生成报告（约 10–30 秒），在线查看或下载 `.txt`

**管理端**
1. 访问 `/kol-intake/admin`
2. 查看所有提交记录列表（昵称、时间、答题数量）
3. 点击记录进入详情，切换「填写内容」/ 「分析报告」两个 Tab
4. 点击「下载文档」获取 `.docx` 格式信息采集表

## 5. 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/bridge` | POST | AI 过渡语生成，每题回答后调用 |
| `/api/submit` | POST | 接收全部答案，调用 AI 生成报告，保存 JSON，返回 `{ ok, id, report }` |
| `/api/submissions` | GET | 返回所有提交记录摘要列表 |
| `/api/submissions/[id]` | GET | 返回单条提交完整数据（答案 + 报告） |
| `/api/download/[id]` | GET | 生成并下载 `.docx` 信息采集表 |

## 6. 环境变量

```env
YUNWU_API_KEY=        # 云雾 AI API 密钥
YUNWU_BASE_URL=       # 云雾 AI 接口地址（默认 https://api.yunwu.ai/v1）
```

**运行：**
```bash
npm install && npm run dev -p 3007
```

访问地址：`http://localhost:3007`（部署时 basePath 为 `/kol-intake`，端口 3007）

## 7. 开发注意事项

- `basePath` 为 `/kol-intake`，`output: 'standalone'` 构建独立 Node.js 服务包
- 报告生成使用 `claude-opus-4-6` + `extended thinking`（budget_tokens: 6000），耗时较长
- 提交数据存储在服务器本地 `data/` 目录，迁移时需同步复制该目录
- `data/` 目录不随代码打包，首次部署时会自动创建
- 问题配置硬编码在 `lib/questions.ts`，修改题目须改代码后重新构建

## 8. 当前状态 / 已知问题

- **完成度**：核心采集和管理功能完整，V2.0 新增工具
- **已知问题**：
  - **Admin 页无鉴权**：任何人可访问 `/kol-intake/admin` 查看全部提交记录，生产环境需添加访问控制
  - 本地文件存储：重新部署时数据丢失，需挂载持久化卷或迁移 `data/` 目录
  - 报告生成无超时提示，AI 响应较慢时用户无进度反馈
  - 题目配置硬编码，非技术人员无法自助修改
