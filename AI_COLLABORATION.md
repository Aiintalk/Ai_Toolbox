# AI 项目开发协作流程

为了方便项目快速推进、版本统一、以及后续维护管理，特制定本协作流程。本流程适用于 Ai_Toolbox 项目的所有功能模块开发，所有开发人员（包括 AI）均需按照本流程执行。

---

## 流程概览

| 阶段 | 步骤 | 执行者 |
|------|------|--------|
| 开发前准备 | 第一步：检查开发环境 | 🤖 AI |
| 开发前准备 | 第二步：拉取最新代码 | 🤖 AI |
| 开发前准备 | 第三步：创建任务分支 | 🤖 AI |
| 开发前准备 | 第四步：读取项目文档，了解全貌 | 🤖 AI |
| 开发前准备 | 第四步确认：选择本次开发模块 | 👤 用户 |
| 开发前准备 | 第五步：读取功能模块文档，记录版本基准 | 🤖 AI |
| 开发前准备 | 第五步确认：确认模块信息和任务需求 | 👤 用户 |
| 功能开发 | 第六步：开始开发 | 🤖 AI |
| 功能开发 | 第七步：开发完成，输出改动摘要请求确认 | 🤖 AI |
| 功能开发 | 第七步确认：确认是否提交 | 👤 用户 |
| 提交推送 | 第八步：更新文档 + 提交代码 | 🤖 AI |
| 提交推送 | 第八步确认：确认是否推送 PR | 👤 用户 |
| 提交推送 | 第九步：推送远程并创建 PR | 🤖 AI |
| 审核合并 | 第十步：等待审核，处理修改意见 | 🤖 AI 等待 / 👤 用户审核 |
| 审核合并 | 第十步确认：PR 合并后告知 AI | 👤 用户 |
| 收尾清理 | 第十一步：清理分支，完成任务 | 🤖 AI |

---

## 详细流程说明

### 🚀 启动方式

用户只需对 AI 说一句话：

> **"请读取项目的 README 文件"**

AI 将自动按以下流程执行，用户在需要确认的节点回复即可。

---

### 第一步：检查开发环境
> 🤖 AI 执行，用户无需操作

AI 依次运行以下命令，确认环境是否就绪：

```bash
git --version          # 检查 Git 是否安装
git config user.name   # 检查 Git 用户名
git config user.email  # 检查 Git 邮箱
git remote -v          # 检查仓库远程地址
gh --version           # 检查 GitHub CLI 是否安装
gh auth status         # 检查 GitHub CLI 是否已登录
node --version         # 检查 Node.js 是否安装
```

**异常处理：**

| 检查项 | 异常情况 | Windows 处理 | Mac 处理 |
|--------|----------|--------------|----------|
| `git --version` 报错 | Git 未安装 | 下载 git-scm.com | `brew install git` |
| 用户名 / 邮箱为空 | 提交信息无署名 | `git config --global user.name "名字"` | 同左 |
| 远程地址不对 | 仓库目录错误 | 切换到正确目录 | 同左 |
| `gh --version` 报错 | GitHub CLI 未安装 | `winget install GitHub.cli` | `brew install gh` |
| `gh auth status` 未登录 | 无法创建 PR | `gh auth login` | 同左 |
| `node --version` 报错 | Node.js 未安装 | 下载 nodejs.org | `brew install node` |

任何一项异常，AI 停止操作并告知用户。

---

### 第二步：拉取最新代码
> 🤖 AI 执行，用户无需操作

```bash
git checkout main
git pull origin main
```

确保本地代码与 GitHub 最新版本一致，避免基于旧代码开发。

---

### 第三步：创建任务分支
> 🤖 AI 执行，用户无需操作

```bash
git checkout -b 姓名缩写月份日期
# 示例：git checkout -b yh0419
```

完成后 AI 输出：

```
✅ 开发环境已准备完成

- 当前分支：yh0419
- 基于：main 最新代码
- 分支已创建完成，可以开发啦！
```

---

### 第四步：读取根目录文档，了解项目全貌
> 🤖 AI 执行 → 👤 用户确认

AI 读取根目录 `README.md` 和 `CHANGELOG.md`，输出项目概况：

```
📋 项目概况

- 项目名称：Ai_Toolbox
- 当前模块：portal / benchmark-analyzer / persona-writer-web / qianchuan-writer-web / seeding-writer-web
- 最近更新：（来自根目录 CHANGELOG.md 的最新一条记录）

请问本次需要开发哪个功能模块？
```

**👤 用户操作：回复本次要开发的功能模块名称**

---

### 第五步：读取功能模块文档，记录版本基准
> 🤖 AI 执行 → 👤 用户确认

AI 读取对应子项目的 `README.md` 和 `CHANGELOG.md`，记录当前版本号，输出模块概况：

```
📦 模块概况

- 当前模块：xxx
- 当前版本：v1.0.0
- 最近更新：（来自子项目 CHANGELOG.md 的最新一条记录）
- 本次任务：（等待用户描述）

确认无误后开始开发。
```

**版本号规则：**

| 改动类型 | 版本变化 | 示例 |
|----------|----------|------|
| 新增功能 | 第二位 +1 | `v1.0.0` → `v1.1.0` |
| 修复问题 | 第三位 +1 | `v1.0.0` → `v1.0.1` |
| 大规模重构 | 第一位 +1 | `v1.0.0` → `v2.0.0` |

**👤 用户操作：确认模块信息，并描述本次任务的具体需求**

---

### 第六步：开始开发
> 🤖 AI 执行，用户无需操作

AI 在当前任务分支中按用户需求完成功能开发。

---

### 第七步：开发完成，请求提交确认
> 🤖 AI 执行 → 👤 用户确认

AI 开发完成后主动输出改动摘要：

```
✅ 功能已开发完成

- 当前模块：xxx
- 本次改动：
  - xxx
  - xxx
- 版本号：v1.0.0 → v1.0.1

是否确认提交？（确认后我将更新 CHANGELOG、提交代码并创建 PR）
```

**👤 用户操作：回复"确认" / "可以" / "没问题"**

用户确认后，AI 自动：
1. 更新子项目 `CHANGELOG.md`（写入新版本号和改动记录）
2. 如有需要同步更新子项目 `README.md`

---

### 第八步：提交代码
> 🤖 AI 执行 → 👤 用户确认

AI 执行：

```bash
git status                          # 查看改动文件
git add 文件1 文件2 文件3            # 只 add 本次相关文件
git commit -m "feat: xxx 改动说明"  # 提交
```

提交完成后告知用户：

```
✅ 代码已提交

- 提交分支：yh0419
- 提交文件：（列出文件）
- 提交说明：feat: xxx 改动说明

即将推送到远程并创建 PR，是否继续？
```

**👤 用户操作：回复"可以" / "继续"**

---

### 第九步：推送远程并创建 PR
> 🤖 AI 执行，用户无需操作

```bash
git push -u origin <当前分支名>
gh pr create --title "..." --base main --head <当前分支名> --body "..."
```

PR 创建成功后告知用户：

```
✅ PR 已创建完成

- 分支：yh0419 → main
- PR 地址：https://github.com/Aiintalk/Ai_Toolbox/pull/xxx
- 请负责人审核后合并。
```

---

### 第十步：等待审核
> 🤖 AI 等待，👤 用户去 GitHub 审核合并

AI 进入等待状态，不做任何操作。

**如果收到修改意见：**
- AI 在原分支修改后重新 push，PR 自动更新
- AI 告知用户"已根据意见修改，请重新审核"

**👤 用户操作：在 GitHub 上审核 PR，合并后告知 AI "已合并"**

---

### 第十一步：清理分支，完成任务
> 🤖 AI 执行，用户无需操作

收到用户"已合并"确认后，AI 执行：

```bash
git checkout main
git pull origin main
git branch -d <当前分支名>
git push origin --delete <当前分支名>
```

清理完成后输出：

```
✅ 分支已清理完成

- 已删除本地分支：yh0419
- 已删除远程分支：yh0419
- 本地 main 已同步最新代码
- 本次任务全部完成 🎉
```

---

## 用户操作汇总

整个流程中，用户只需操作以下 **4 个节点**：

| 节点 | 用户需要做什么 |
|------|----------------|
| 第四步完成后 | 告诉 AI 本次开发哪个功能模块 |
| 第五步完成后 | 确认模块信息，描述本次任务需求 |
| 第七步完成后 | 确认是否提交（回复"确认/可以/没问题"） |
| 第八步完成后 | 确认是否推送 PR（回复"可以/继续"） |
| 第十步完成后 | 去 GitHub 合并 PR，完成后告知 AI"已合并" |

---

## 注意事项

- 每次开发必须从最新 `main` 拉取代码，不得基于旧代码开发
- 一个分支只做一件事，不允许混入多个需求
- 版本号必须基于子项目 `CHANGELOG.md` 中的最新版本，不得自行假设
- 提交时只 add 本次相关文件，禁止 `git add .` 全量提交
- PR 未合并前不得删除分支
- 如有新需求在 PR 审核期间到来，新开分支处理，不得写入当前分支
